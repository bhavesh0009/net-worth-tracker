# Product Requirement Document (PRD): Net Worth Tracker Application

**Version:** 1.0
**Date:** December 01, 2025
**Platform:** Web Application
**Tech Stack:** React (Vite/Next.js) + Google Cloud BigQuery

---

## 1. Project Overview
**Objective:**
Build a lightweight, personal web application to replace a manual Excel-based net worth tracker. The goal is to transition from "monthly aggregate rows" to "account-level snapshots," allowing for granular historical analysis while maintaining a simple, high-speed user interface.

**Target User:**
Single User (Personal Finance).

**Key Constraint:**
Database complexity must be kept to a minimum (Max 3 Tables) to ensure ease of maintenance.

---

## 2. Core Functional Requirements

### 2.1. Account Management (Setup)
* **Create/Edit:** User must be able to define "Accounts" where money is stored.
* **Categorization:** Each account must have a type (Asset/Liability) and a category (e.g., Real Estate, Equity, EPF).
* **Active Status:** User can mark accounts as "Inactive" to hide them from future data entry lists without deleting historical data.

### 2.2. The "Monthly Run" (Data Entry)
* **Trigger:** User initiates a new "Period" (e.g., November 2025).
* **Global Comment:** Ability to add a high-level note for the month (e.g., "Bonus received," "Market correction").
* **Pre-fill Feature:** When creating a new month, the input fields should pre-fill with the *previous month's balances*. This drastically reduces data entry time—the user only edits the changed figures.
* **Calculated Feedback:** As the user types, the UI should display a live "Total Net Worth" calculation to verify against expectations before saving.

### 2.3. Dashboard & Analytics
* **Current Snapshot:** Display total Assets, total Liabilities, and Net Worth.
* **Historical Trend:** A line chart showing Net Worth growth over time.
* **Granularity:** (Future Scope) Ability to filter the trend line by category (e.g., "Show me Net Worth without Real Estate").

## 3. Database Schema (BigQuery)
*Constraint: Strictly limited to 3 tables for simplicity.*

### Table 1: `accounts`
*Stores the static list of financial accounts.*

| Column Name | Type | Description |
| :--- | :--- | :--- |
| `id` | `STRING` | Primary Key (UUID generated by application) |
| `name` | `STRING` | e.g., "Angel One PF", "HDFC Home Loan" |
| `type` | `STRING` | "Asset" or "Liability" |
| `category` | `STRING` | e.g., "Real Estate", "Equity", "Cash" |
| `is_active` | `BOOLEAN` | Default `true`. Set `false` to archive. |

### Table 2: `periods`
*Represents the time buckets (the monthly columns).*

| Column Name | Type | Description |
| :--- | :--- | :--- |
| `id` | `STRING` | Primary Key (UUID generated by application) |
| `month_date` | `DATE` | e.g., `2025-11-01` |
| `notes` | `STRING` | Global comment for the month. |
| `total_nw` | `NUMERIC` | Cached total net worth (Optional, for speed). |

### Table 3: `records`
*The transactional snapshots. Intersection of an Account and a Period.*

| Column Name | Type | Description |
| :--- | :--- | :--- |
| `id` | `STRING` | Primary Key (UUID generated by application) |
| `period_id` | `STRING` | FK linked to `periods.id` |
| `account_id` | `STRING` | FK linked to `accounts.id` |
| `amount` | `NUMERIC` | The balance at that specific time. |

## 4. Frontend Design & UI/UX Strategy

**Design Philosophy:** Clean, "Financial Dashboard" aesthetic. High contrast for numbers.
**Recommended Framework:** React + Tailwind CSS.

### 4.1. Navigation
* **Sidebar/Top Bar:** Simple links: "Dashboard", "Add Month", "Manage Accounts".

### 4.2. Page: The Dashboard (Home)
* **Top Section (Cards):**
    * **Net Worth:** Big bold font. Green/Red indicator showing change vs previous month.
    * **Asset Total:** Displayed in Blue.
    * **Liability Total:** Displayed in Red.
* **Middle Section (Visuals):**
    * **Main Chart:** Area chart showing Net Worth history (X-axis: Date, Y-axis: ₹).
    * **Allocation:** Simple Pie chart (Real Estate vs Equity vs Debt).
* **Bottom Section (Table):**
    * List of current account balances sorted by value.

### 4.3. Page: Add Month (The "Run")
* **Header:** Date Picker (Month/Year) + Comment Input.
* **The Grid:**
    * A clean list of all active accounts.
    * Input boxes next to each name.
    * **Crucial UX:** Auto-focus the first input. Allow `Tab` key to move to the next input rapidly.
* **Footer:** "Save Snapshot" button.

## 5. Implementation Roadmap

1.  **Phase 1 (Backend):** Set up Google Cloud BigQuery project, create dataset, and create the 3 tables with the schema defined above.
2.  **Phase 2 (Settings UI):** Build the page to Add/Edit `accounts`.
3.  **Phase 3 (Entry UI):** Build the "Add Month" flow with pre-fill logic (fetch `previous_month` data).
4.  **Phase 4 (Dashboard):** Build the charts and summary cards.
5.  **Phase 5 (Legacy Data):** User to manually enter the "Current Month" to initialize the system.
